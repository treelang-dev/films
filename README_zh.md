## 💡 项目背景 (Why Films?)

对于票务从业者或数据分析师而言，获取一张**淘票票**优惠券所支持的全国影院完整列表是一项至关重要的任务。然而，官方平台并未提供任何批量导出或跨城市搜索的功能。传统方法不仅需要耗时费力地在PC端手动查询，或者依赖于复杂的、容易被封禁的爬虫脚本。

**Films** 正是为了解决这一痛点而生。它是一个“一劳永逸”的全自动化安卓工具。您只需输入优惠券的 `fcode`，应用便会自动完成以下所有工作：
1.  自动发现淘票票支持的所有城市列表。
2.  遍历每一个城市，抓取适用影院数据。
3.  将全国范围的数据进行汇总。
4.  将最终的干净数据导出为一个结构清晰的 `.xlsx` Excel 文件。

它将一个原本需要数小时手动操作的任务，转变成了在手机上的一键式后台处理。

## ✨ 主要特性 (Features)

*   **📲 全自动化流程**: 一键启动，应用将自动完成城市发现、数据抓取和结果聚合的全过程。
*   **🌍 全国范围覆盖**: 自动抓取平台上的每一个城市，确保数据集的完整性。
*   **📄 Excel 导出**: 使用 Apache POI 库直接生成格式规范的 `.xlsx` 文件，包含“城市”、“影院名称”和“地址”列。
*   **⚙️ WebView 核心**: 通过一个隐藏的 WebView 来渲染页面并执行 JavaScript，能够高度模拟真实用户行为，有效降低被屏蔽的风险。
*   **📊 实时进度反馈**: 界面清晰地展示当前状态、进度条和关键日志，让您随时掌握抓取进程。
*   **⚠️ 可靠的错误处理**: 内置超时和失败城市重试逻辑，即使在网络不稳定的情况下也能确保任务的最终完成。

## 🛠️ 技术实现深度解析

本应用的核心是围绕一个由状态机（`IDLE`, `FETCHING_CITIES`, `FETCHING_CINEMAS`）控制的隐藏 WebView 实例来运作的。

1.  **第一阶段：城市列表发现**
    *   首先，加载一个样本城市（如上海）的优惠券页面。
    *   通过注入的 JavaScript，代码模拟用户点击“城市选择”按钮的行为。
    *   等待城市列表在 DOM 中完全加载后，提取全国所有城市的名称和城市代码。
    *   提取到的数据通过 `JavascriptInterface` 从 WebView 的 JavaScript 环境回传到原生 Java 代码中。

2.  **第二阶段：全国影院数据抓取**
    *   应用随即进入一个循环，遍历第一阶段获取到的所有城市。
    *   对每个城市，构造其专属的URL并加载页面。
    *   页面加载完毕后，注入第二段 JavaScript 负载，用以查找并提取页面上所有影院的名称和地址。
    *   如果某个城市没有适用影院或页面为空，程序会优雅地跳过并继续下一个城市。

3.  **第三阶段：数据聚合与导出**
    *   所有抓取到的影院信息都被缓存在内存中。
    *   当所有城市遍历完毕，应用会调用 **Apache POI** 库来动态创建一个 Excel 工作簿。
    *   最后，通过安卓的存储访问框架（Storage Access Framework），应用会唤起文件保存对话框，让用户选择最终 `.xlsx` 文件的存储位置。

## 🚀 开始使用 (开发者)

按照以下步骤，即可在本地编译并运行此项目。

### 环境要求

*   Android Studio (Arctic Fox 或更高版本)
*   安卓真机或模拟器 (API 21+)

### 安装步骤

1.  克隆本仓库：
    ```sh
    git clone https://github.com/your-username/films.git
    ```
2.  在 Android Studio 中打开此项目。
3.  等待 Gradle 自动同步并下载所需依赖项，主要是 `org.apache.poi`。
4.  构建并运行项目。请注意，应用正常工作需要 `INTERNET` 网络权限。

## 🧑‍💻 如何使用本应用

1.  获取您想要分析的淘票票优惠券的 `fcode`。
2.  打开 **Films** 应用。
3.  在输入框中填入 `fcode`。
4.  点击 **“Start Scraping”** (开始抓取) 按钮。
5.  等待程序自动执行。您可以在屏幕上观察到实时进度。
6.  任务完成后，应用将提示您保存生成的 `all_city_cinemas.xlsx` 文件。
